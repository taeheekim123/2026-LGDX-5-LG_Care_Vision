## 결론

`Chat.tsx`와 `SelfCare.tsx`는 병합 전 디자인 기준(`7da5d52`)과 현재 `heejin` 상태가 다릅니다.

특히 `Chat.tsx`는 병합 과정에서 병합 전의 글래스 UI, 전환 애니메이션, 인라인 매뉴얼 확장 방식, 일부 색상 톤이 현재 버전으로 덮인 상태입니다. `SelfCare.tsx`도 병합 전 제품 카드/영상 카드 레이아웃이 일부 바뀌었습니다.

다만 현재 버전에는 Taehee 브랜치에서 들어온 AR/TTS 연동용 데이터 전달 로직이 추가되어 있습니다. 따라서 예전 파일을 통째로 되돌리면 AR/TTS 기능이 다시 빠질 수 있어, 디자인만 선별 복구하는 방식이 안전합니다.

## Chat.tsx 검증

확인 결과, 병합 전 디자인이 현재 파일에 그대로 남아 있지 않습니다.

병합 전에는 채팅 저장 키가 다음처럼 최신 전환 UI 기준이었습니다.

- 병합 전: `CHAT_STORAGE_KEY = "chat_messages_v20260618_transition_v2"`
- 현재: `CHAT_STORAGE_KEY = "chat_messages_v20260612"`

병합 전에는 다음 디자인/상태 요소가 있었습니다.

- `expandedManualGuideIds`: 매뉴얼 가이드를 채팅 안에서 펼쳐 보여주는 상태
- `shouldRenderChatLayer`: 초기 화면과 채팅 화면 전환 렌더링 제어
- `chatGlassSurface`, `chatSoftCard`, `chatAccentSurface`: 글래스 카드/채팅 말풍선 스타일
- 헤더의 `Online - Support ready` 상태 표시
- 보라색 계열 액센트(`#8b80e8`, `#9b8ef6`) 기반 서비스 카드

현재 파일에서는 위 요소 중 일부가 제거되거나 단순화되었습니다.

- 채팅 헤더가 `LG Chat + 상태 표시` 구조에서 `Chat` 중심의 단순 헤더로 바뀜
- 봇 말풍선/옵션 버튼이 글래스 UI에서 흰색/빨간색(`#ff4c49`) 계열 UI로 바뀜
- `Confirm Request Information` 카드 색상이 보라색 계열에서 빨간색 계열로 바뀜
- 매뉴얼 가이드가 `expandedManualGuideIds` 기반 확장 방식이 아니라 현재 방식으로 표시됨

유지된 기능도 있습니다.

- 제품 선택 흐름은 남아 있음
- 제품군 선택 흐름은 남아 있음
- 모델 선택 흐름은 남아 있음
- `canOpenArGuide`, `arGuideStepsFromOptions` 같은 AR 이동/단계 전달 로직은 현재 버전에 추가되어 있음
- `display_steps`, `tts_text`, `audio_url`, `targetHint`, `targetClasses` 등 AR/TTS용 데이터 전달 구조가 추가되어 있음

## SelfCare.tsx 검증

`SelfCare.tsx`도 병합 전 디자인과 현재 디자인이 다릅니다.

병합 전 제품 카드 쪽에는 다음 구조가 있었습니다.

- 전체 컨테이너: `px-[18px] pt-[40px]`
- 제품 이미지 상단의 작은 배지: `LG Whisen Wall-mounted`
- 제품 카드 내부 상단 여백: `pt-[24px]`
- `Product Type`, `Registered Date`가 중앙 정렬형 정보 블록으로 표시됨

현재 파일에서는 다음처럼 변경되었습니다.

- 카드 좌우 여백이 `mx-6` 중심으로 바뀜
- 제품 카드의 작은 상단 배지 구조가 제거되고, 모델명이 일반 텍스트로 내려옴
- `Product Type`, `Registered Date` 표시 방식이 좌우 정렬형으로 바뀜
- 영상 카드에서 병합 전 `Official Video Guide` 제목 블록이 빠지고, 단순 영상 박스와 확대 아이콘(`Maximize2`) 구조로 바뀜

반대로 현재 파일에 추가된 기능도 있습니다.

- `AR_GUIDE_STEP_TITLES`
- `AR_GUIDE_STEP_TARGETS`
- `arGuideStepsFromOptions`
- `handleStartArGuide`
- `display_steps`, `tts_text`, `audio_url`, `targetHint`, `targetClasses` 기반 AR 가이드 전달

즉, Self Care는 디자인이 일부 후퇴했지만 AR/TTS 연결용 기능은 현재 버전에 더 많이 들어와 있습니다.

## 복구 방향

권장 복구 방식은 통째 체크아웃이 아니라 선별 병합입니다.

1. `Chat.tsx`
   - 병합 전의 글래스 말풍선, 헤더, 애니메이션, `expandedManualGuideIds` 기반 인라인 매뉴얼 UX를 복구
   - 현재 추가된 `canOpenArGuide`, `arGuideStepsFromOptions`, `display_steps`, TTS/AR target 데이터는 유지
   - 채팅 저장 키는 병합 전 디자인 기준인 `chat_messages_v20260618_transition_v2`로 되돌리는 것이 좋음

2. `SelfCare.tsx`
   - 병합 전 제품 카드, 영상 카드, 여백/타이포 구조를 복구
   - 현재 추가된 `handleStartArGuide`, `AR_GUIDE_STEP_TARGETS`, `display_steps` 기반 AR/TTS 전달 로직은 유지

3. 주의할 점
   - `git checkout 7da5d52 -- Chat.tsx SelfCare.tsx`처럼 통째 복구하면 Taehee 브랜치에서 들어온 AR/TTS 연동 기능이 빠질 수 있음
   - 현재는 기능 추가분과 디자인 복구분을 같이 살리는 수동 병합이 필요함
   - 채팅은 `localStorage` 저장 키가 바뀌면 이전 화면 상태가 남아 보일 수 있으므로, 복구 후 브라우저 저장 데이터를 초기화하거나 저장 키를 새 버전으로 올리는 것이 좋음

## 현재 판단

사용자가 말한 “병합 전 디자인이 사라졌다”는 판단은 맞습니다.

다만 현재 파일은 단순히 예전 버전으로 돌아간 것이 아니라, Taehee 브랜치의 AR/TTS 연동 기능이 섞이면서 디자인 일부가 덮인 상태입니다. 다음 작업은 `Chat.tsx`와 `SelfCare.tsx`를 병합 전 디자인으로 되살리되, 현재 들어온 AR/TTS 기능은 유지하는 방식으로 진행하는 것이 맞습니다.

- 병합 전 `heejin` 디자인 기준인 글래스 UI, 채팅 전환 애니메이션, `LG Chat` 헤더, `Online - Support ready` 표시를 복구했습니다.

- 병합 전 디자인의 인라인 `Manual Guide` 확장 방식(`expandedManualGuideIds`)을 복구했습니다.
- `taehee` 쪽 기능인 AR 이동 가능 여부 판단(`canOpenArGuide`)을 유지했습니다.

- 병합 전 디자인의 `Official Video Guide` 카드 헤더와 영상 카드 스타일을 복구했습니다.
- `taehee` 쪽 기능인 AR 시작 시 단계별 데이터 전달(`handleStartArGuide`)을 유지했습니다.
- `display_steps`, TTS 텍스트, 오디오 URL, AR target 정보를 `/ar-guide`로 넘기도록 유지했습니다.
