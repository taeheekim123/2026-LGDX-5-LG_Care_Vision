## Chat 화면

### 선택 버튼 스타일 정리

- 일반 선택 버튼(`Troubleshoot Product Issue`, `Appliance Care Guide`, `Connect to Service Center` 등)을 카드형 outline 스타일로 정리했다.
- 기본 상태는 흰 카드, 보라 글자, 보라 외곽선으로 맞췄다.
- hover 상태는 흰색 위에 보라색이 살짝 올라오는 배경과 더 진한 외곽선을 적용했다.
- selected 상태는 진한 보라/블루 그라데이션과 흰 글자로 표시되도록 했다.
- 문제 상태 선택 버튼도 동일한 버튼 상태 규칙을 적용했다.

### 모델 선택 버튼 스타일 정리

- `Select the registered appliance that needs service.` 아래 모델 선택 버튼에도 보라 계열 outline/hover/selected 상태를 적용했다.
- 기본 상태는 흰 카드와 보라 외곽선, 보라 제품명으로 표시된다.
- hover 상태는 흰색 베이스 위에 보라 레이어가 올라오도록 조정했다.
- selected 상태는 진한 보라/블루 그라데이션과 흰 글자로 표시된다.

### 방문 일정 선택 버튼 정리

- `Select Visit Date`에서 선택하지 않은 날짜가 선택된 것처럼 보이는 문제를 줄였다.
- 새 예약 카드가 뜰 때 `selectedDate`와 hover 상태를 초기화하도록 했다.
- 날짜 버튼은 선택된 경우에만 진한 선택 색상이 적용된다.
- `Select Visit Time` 버튼에도 보라 글자와 보라 외곽선을 적용했다.
- 시간 버튼 hover도 투명하게 비치는 방식이 아니라 흰색 위에 보라색이 올라오는 방식으로 맞췄다.

### 전환 스크롤 개선

- `Please select the current status.` 메시지가 뜰 때 이전 채팅 내용이 위에 보이는 문제를 줄였다.
- `scrollIntoView({ block: "start" })` 대신 채팅 스크롤 컨테이너 안에서 직접 위치를 계산해 이동하도록 바꿨다.
- 서비스센터 플로우에서는 `Select the registered appliance that needs service.` 메시지부터 같은 포커스 스크롤 처리를 적용했다.
- 모델 선택 메시지가 뜰 때 아래 여백을 확보하고, 모델 선택 후 여백을 해제하도록 했다.

## Device / Self Care 화면

### My Devices 카드 조정

- `My Devices`의 제품 카드 텍스트가 크게 보이는 문제를 줄였다.
- 제품명 크기를 `16px`에서 `15px`로 낮췄다.
- 모델명은 `12px` 기준으로 유지했다.
- 상태 배지는 `11px`에서 `10px`로 낮췄다.
- 카드 높이와 제품 이미지 크기도 조금 줄여 `Settings > Edit Profile` 카드 톤에 더 가깝게 맞췄다.

### Device Detail 제품명 조정

- `/device/1` 상세 화면의 제품명 영역을 `16px`에서 `15px`로 낮췄다.
- 대상 위치: `frontend/src/app/pages/DeviceDetail.tsx`

### Self Care 제품명 조정

- `/self-care` 화면의 `Living Room Air Conditioner` 제품명을 `16px`에서 `15px`로 낮췄다.
- 대상 위치: `frontend/src/app/pages/SelfCare.tsx`
