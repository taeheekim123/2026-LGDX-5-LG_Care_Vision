## Render/Supabase 환경 API PostgreSQL region lookup 보정

### 요청/배경

- Render 프론트 배포 후 Home 화면의 Fine Dust 값이 `10`으로 표시됨.
- live 환경 API 확인 결과 실제 미세먼지 값이 10인 것이 아니라, 환경 관측값이 `null`로 내려와 프론트 fallback 기본값 10이 표시되고 있었음.
- `GET /api/v1/environment/current?region=India&city=Hyderabad` 응답에서 `mode=fallback_cache`, `aqi=null`, `pm25=null`, `pm10=null`을 확인함.

### 원인

- 환경 API refresh 중 `create_environment_observation -> resolve_region_id` 경로에서 PostgreSQL/Supabase가 아래 SQL 조건의 파라미터 타입을 추론하지 못함.

```text
(? IS NULL OR city = ?)
```

- Render live 오류:

```text
psycopg.errors.IndeterminateDatatype
could not determine data type of parameter
```

### 수정 파일

```text
backend/app/repositories/sqlalchemy_repositories.py
backend/tests/test_environment_data_adapter.py
docs/ar/CareShot_AR_전체개발순서.md
docs/ar/development_logs/2026-06-19_개발로그.md
```

### 구현 내용

- `resolve_region_id`에서 city 조건을 nullable predicate로 처리하지 않도록 변경함.
- city가 있으면 SQL에 `AND city = ?`를 직접 추가함.
- city가 없으면 city 조건을 붙이지 않음.
- PostgreSQL/Supabase에서 타입 추론이 필요한 `? IS NULL` 조건을 제거함.

## PR #22 보류 및 self A/S aircon context detector 후속 작업 전환

### 요청/배경

- self care 필터청소 AR에서 에어컨 탐지 부분만 self A/S 모델의 aircon detector를 재사용하는 구현을 PR #22로 올렸으나, 실제 모바일 카메라/실물 탐지 성능 검증 전이라 반영을 보류하기로 했다.
- 사용자가 PR #22를 닫고 `taehee` 브랜치를 이전 상태로 되돌린 뒤, 해당 아이디어는 나중에 할 일로 문서에 남기라고 요청했다.

### 처리 내용

```text
- 로컬 taehee 브랜치를 PR #22 이전 기준인 origin/main(58dee92) 상태로 되돌림.
- ARGuide.tsx 구현 변경은 제거함.
- smoke script 수정도 제거함.
- 구현 대신 전체개발순서 문서에 후속 작업 후보로 기록함.
```

### 후속 작업으로 남긴 방향

```text
procedure_type == filter_cleaning일 때:
1. self_as_no_cooling 모델 프로필로 ["aircon", "indoor_unit"]만 먼저 탐지
2. aircon context가 확인된 경우에만 self_care 모델로 ["filter", "front_filter", "front_cover"] 탐지
3. self A/S 모델 결과는 필터 overlay 직접 기준으로 사용하지 않음
4. filter 미검출 시 aircon bbox + reference part map fallback을 별도 구현
```

### 보류 이유

```text
- 로컬 자동 검증은 코드 흐름/payload 확인 수준이다.
- 실제 탐지 성능, confidence, bbox 안정성은 모바일 카메라와 실제 에어컨 또는 reference image로 검증해야 한다.
- 발표 환경에서 잘 잡히는지 확인 전에는 main 반영하지 않는 것이 안전하다.
```
