# 2026-06-19 개발로그

## Render/Supabase 환경 API PostgreSQL region lookup 보정

### 요청/배경

- Render 프론트 배포 후 Home 화면의 Fine Dust 값이 `10`으로 표시됨.
- live 환경 API 확인 결과 실제 미세먼지 값이 10인 것이 아니라, 환경 관측값이 `null`로 내려와 프론트 fallback 기본값 10이 표시되고 있었음.
- `GET /api/v1/environment/current?region=India&city=Hyderabad` 응답에서 `mode=fallback_cache`, `aqi=null`, `pm25=null`, `pm10=null`을 확인함.

### 원인

- 환경 API refresh 중 `create_environment_observation -> resolve_region_id` 경로에서 PostgreSQL/Supabase가 아래 SQL 조건의 파라미터 타입을 추론하지 못함.

```text
(? IS NULL OR city = ?)
```

- Render live 오류:

```text
psycopg.errors.IndeterminateDatatype
could not determine data type of parameter
```

### 구현 내용

- `resolve_region_id`에서 city 조건을 nullable predicate로 처리하지 않도록 변경함.
- city가 있으면 SQL에 `AND city = ?`를 직접 추가함.
- city가 없으면 city 조건을 붙이지 않음.
- PostgreSQL/Supabase에서 타입 추론이 필요한 `? IS NULL` 조건을 제거함.
