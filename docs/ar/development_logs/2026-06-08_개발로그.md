## OpenWeather/WAQI 실제 API key 연결 검증

- 요청 내용:
  - `.env`에 입력된 API key가 실제 API 호출에 연결되는지 확인
- 사전 확인:
  - `OPENWEATHER_API_KEY`: 설정됨, 값 길이 32
  - `WAQI_API_KEY`: 설정됨, 값 길이 40
  - key 원문은 보안상 출력하지 않음
- 검증 방식:
  - FastAPI `TestClient`로 실제 `/api/v1/environment/refresh` endpoint 호출
  - provider별 `force_refresh=true`, `cache_ttl_minutes=0`으로 cache가 아닌 외부 API refresh 강제
  - 검증 provider:
    - `ENV_PROVIDER_OPENWEATHER`
    - `ENV_PROVIDER_WAQI`
- OpenWeather 검증 결과:
  - HTTP status: 200
  - fetch status: `success_external_api`
  - response_summary.mode: `external_api_refresh`
  - runtime_provider_id: `ENV_PROVIDER_OPENWEATHER`
  - observation_id: `ENVOBS_B15D77795783`
  - 수신 값 예시: temperature 35.7, humidity 27, AQI 125, PM2.5 32.9, PM10 81.31
- WAQI 검증 결과:
  - HTTP status: 200
  - fetch status: `success_external_api`
  - response_summary.mode: `external_api_refresh`
  - runtime_provider_id: `ENV_PROVIDER_WAQI`
  - observation_id: `ENVOBS_ACFF454C372C`
  - 수신 값 예시: AQI 81, PM2.5 34, PM10 81
- 판단:
  - `.env` key 로드와 실제 외부 API 성공 응답 검증 완료
  - 2026-06-05에 남아 있던 “실제 OPENWEATHER_API_KEY, WAQI_API_KEY 발급 후 실 API 성공 케이스 운영 검증” 과제는 현재 실행 기준으로 완료 처리 가능

  - WAQI 응답은 AQI/PM 중심이며 temperature/humidity는 응답 지점에 따라 null일 수 있음
