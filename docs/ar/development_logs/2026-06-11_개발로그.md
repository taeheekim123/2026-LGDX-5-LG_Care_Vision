## 20. LLMServiceMock 또는 LLM adapter 구현

요청:

```text
개발 단계에서는 비용 없는 mock/로컬 rule adapter를 우선 사용.
최종 구조에서는 OpenAI 등 외부 LLM API로 교체 가능한 interface 작성.
LLM 역할은 고객 문의 요약, slot 추출 보조, 다국어 응답 문장 생성,
안전 문구 템플릿 변환으로 제한.
공식근거 판단 자체는 RAGService/DecisionEngineV2가 담당.
```

수정:

```text
04_백엔드/app/llm_service.py
- LLMService protocol 추가
- LLMServiceMock 추가
- ExternalLLMAdapter placeholder 추가
- create_llm_service(provider) factory 추가
- LLM_PROMPT_SCHEMA, LLM_OUTPUT_SCHEMA 추가
- mock provider 기본값: CARESHOT_LLM_PROVIDER 미설정 시 mock_rule_adapter 사용
- allowed task: inquiry summary, slot candidate hint, response draft, safety copy transform
- forbidden task: final AR permission, official evidence verification, high-risk override, unverified repair step generation

04_백엔드/app/services.py
- CareShotBackendService에 self.llm_service = create_llm_service() 추가

04_백엔드/app/chatbot_engine.py
- ChatbotEngine이 self.service.llm_service.assist_chat_turn() contract만 호출하도록 연결
- /api/v1/chat/messages 응답에 chatbot_engine.llm_assist 추가
- /api/v1/chat/messages 응답에 chatbot_engine.llm_policy 추가
- analysis.llm_assist에도 동일 assist 결과를 포함
- ai_message 문구는 mock LLM response_text를 우선 사용하되, 최종 decision/action은 기존 DecisionEngine/RAG/Safety 결과를 유지

04_백엔드/tests/test_llm_service_mock.py
- prompt/output schema contract 검증
- LLM slot candidate가 non-authoritative임을 검증
- expert_as high-risk에서 LLM이 safety override를 하지 않음을 검증
- multi-turn clarification copy가 mock LLM response_text로 노출됨을 검증
```

중요 제한:

```text
LLMServiceMock은 최종 AR 허용 여부를 결정하지 않는다.
LLMServiceMock은 공식자료 match를 검증하지 않는다.
LLMServiceMock은 High Risk를 self-guidance로 낮추지 않는다.
최종 gate는 계속 Rule/Safety Guard + RAG 공식근거 + DecisionEngine이 담당한다.
외부 LLM API adapter는 provider 교체 경계만 준비했고, 실제 API 호출은 아직 활성화하지 않았다.
```
