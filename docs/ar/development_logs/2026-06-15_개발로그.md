## 22-9. Care Risk 환경 factor 응답/점수 반영 검증 및 보정

요청:

```text
environment_adapter.get_environment() 결과가 Care Risk API 응답과 score engine에 정상적으로 들어오게 고치고,
Ahmedabad/Gujarat mock 또는 live 환경값 중 습도/AQI/몬순 값이 있으면 trigger_reason[1], [2]까지 생기게 검증한다.
```

사전 확인:

```text
POST /api/v1/care/risk/evaluate
-> environment_observation 응답 포함
-> 최신 live/cache row 기준 humidity=71, AQI=68, monsoon=null
-> trigger_reason[0]: 최근 일평균 사용 시간은 6.0시간입니다.
-> trigger_reason[1]: 현재 습도는 71%로 높습니다.

AQI는 150 미만이라 제외, monsoon은 null이라 제외.
```

확인된 문제:

```text
ENVIRONMENT_OBSERVATION cache row는 monsoon을 top-level monsoon_intensity 필드로 보유한다.
하지만 CareRiskScoreEngine.environment_score()는 payload.rain_monsoon_intensity / payload.monsoon_intensity만 읽고 있었다.
따라서 DB cache row에서 monsoon_intensity가 있어도 몬순 factor가 빠질 수 있었다.
```



실패 및 수정:

남은 한계:

