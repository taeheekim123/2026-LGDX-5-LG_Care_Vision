## 백엔드 팀에 전달할 핵심 요약

현재 프론트는 다음 흐름을 기대한다.

```text
채팅 선택 context
→ POST /api/ai/chat
→ guide_options 응답
→ SelfCare 또는 ARGuide 화면으로 전달
→ GET /api/v1/guides/options
→ AR Guide 단계 표시
→ 필요 시 POST /api/v1/ar/filter-detect
→ 필요 시 POST /api/v1/tts/synthesize
```

백엔드에서 유지해야 하는 응답 핵심 필드:

```text
guide_options.display_title
guide_options.procedure_type
guide_options.display_steps
guide_options.manual_guides
guide_options.youtube_recommendations
guide_options.ar_guides
```

특히 `procedure_type=filter_cleaning`은 현재 프론트의 제품 상세 화면과 AR Guide 기본 흐름에서 사용 중이다.

## 2026-06-19 추가 재검증 결과

### AR 모델

초기 검증에서는 백엔드 venv에 `ultralytics`가 없어 실제 YOLO 모델이 로드되지 않았다.

```text
model_loaded: false
mode: none
message: YOLO model load failed: No module named 'ultralytics'
```

조치:

- `backend/requirements.txt`에 `ultralytics>=8.3` 추가
- `backend/.venv`에 requirements 재설치
- FastAPI 백엔드 재시작

재검증 결과:

```text
POST http://localhost:8000/api/v1/ar/filter-detect

model_loaded: true
mode: yolo
message: null
```

현재 실제 모델 파일:

```text
03_AI로직/models/filter_detection/best.pt
```

즉 현재 로컬 기준 AR 모델 로드는 정상이다. 단, 테스트 입력이 1x1 샘플 이미지라 탐지 박스는 없을 수 있다.

### TTS

현재 AR Guide의 TTS는 Web Speech fallback 기준으로 동작 가능하다.

Google Cloud TTS mp3 직접 합성 API는 코드와 라우터는 연결되어 있으나, 로컬 환경변수가 꺼져 있어 비활성 상태다.

```text
POST http://localhost:8000/api/v1/tts/synthesize
→ 503 Google TTS is disabled
```

Google TTS까지 켜려면 백엔드 실행 환경에 아래 값이 필요하다.

```env
GOOGLE_TTS_ENABLED=1
GOOGLE_TTS_CREDENTIALS_JSON=...
GOOGLE_TTS_PREGENERATE=1
```
