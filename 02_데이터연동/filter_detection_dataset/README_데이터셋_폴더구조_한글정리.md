# 필터 인식 데이터셋 폴더 구조 정리

## 데이터셋 목적

이 폴더는 CareShot AR 가전케어 AI에서 벽걸이 에어컨 필터 위치를 인식하기 위한 이미지 데이터셋 작업 공간이다.

주요 목적은 다음과 같다.

- LG 벽걸이 에어컨 실내기와 필터 영역 이미지 후보 수집
- 필터 bbox 사전 라벨 생성 및 검수
- Roboflow 업로드용 패키지 구성
- Roboflow YOLOv8 내보내기 데이터셋 검증
- YOLO 필터 검출 모델 학습 결과와 FastAPI 배포용 `best.pt` 관리

## 정리 원칙

- 사람이 보는 최상위 작업 폴더와 주요 산출물 폴더는 한글명으로 정리했다.
- 코드, 문서, CSV 매니페스트, 컨택트시트, 로그, 원천 이미지, Roboflow 산출물, 학습 결과를 분리했다.
- YOLO/Roboflow 도구가 직접 참조하는 내부 표준 폴더명 `images`, `labels`, `train`, `valid`, `test`는 변경하지 않았다.
- 이미지 파일명과 라벨 파일명은 1:1 매칭이 중요하므로 대량 한글명 변경 대상에서 제외했다.

## 주의 사항

- `20_Roboflow_업로드패키지` 내부의 `images`, `labels`, `darknet.labels`, `roboflow_upload_manifest.csv` 등은 Roboflow 업로드 흐름과 맞물려 있으므로 표준명을 유지한다.
- `30_Roboflow_내보내기_YOLO` 내부의 `data.yaml`, `train/images`, `train/labels`, `valid/images`, `valid/labels`, `test/images`, `test/labels`는 YOLO 학습 도구가 요구하는 구조이므로 변경하지 않는다.
- 이미지 파일과 라벨 파일은 stem이 같아야 한다. 예를 들어 `sample_001.jpg`와 `sample_001.txt`가 서로 매칭되므로 파일명 일괄 변경은 별도 변환 스크립트 없이 수행하지 않는다.
- 향후 스크립트를 실행할 때는 한글 폴더명 기준으로 경로가 갱신되어 있는지 먼저 확인한다.
